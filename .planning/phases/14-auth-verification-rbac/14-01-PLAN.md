Create Passport.js Google OAuth strategy that:
1. Accepts Google OAuth callback with profile data
2. Checks UserProvider table for existing provider ID match
3. If found, return existing User; if not, check User table by email
4. If User found by email, create UserProvider link without creating duplicate User
5. If no match, create new User with:
   - email from Google profile
   - name from Google profile
   - role = 'USER' (default)
   - isNewUser = true
   Then initialize Wallet record with depositedBalance = 0, earnedBalance = 0, totalDeposited = 0
6. Return authenticated user object
7. Use strict error handling: log linking errors, return meaningful messages

**Code Pattern:**
- Use `GoogleStrategy extends PassportStrategy(Strategy, 'google')`
- Configure scope: ['profile', 'email']
- Call parent validate(accessToken, refreshToken, profile)
- Query prisma.userProvider.findUnique first (provider + providerId)
- If not found, query prisma.user.findUnique by email
- Create link or new user atomically

**Auth-01 compliance:** Strategy accepts and validates Google callback  
**Auth-02 compliance:** Implements legacy account linking by provider ID and email  
**Auth-03 compliance:** Creates new user with default USER role and Wallet  
**Auth-04 compliance:** Note secure cookie handling (will be configured in app.module)

  <action>
Create NestJS role-based access control infrastructure:

**@Roles Decorator (roles.decorator.ts):**
- Simple decorator that stores required roles in metadata: `SetMetadata('roles', roles)`
- Accept variadic args: `@Roles('ADMIN', 'AUTHOR')`
- Export as `export const Roles = (...roles: string[]) => SetMetadata('roles', roles)`

**RolesGuard (roles.guard.ts):**
- Implement `CanActivate` interface
- Constructor: inject `Reflector` for metadata access
- canActivate method:
  1. Get required roles from route metadata using `reflector.get('roles', context.getHandler())`
  2. If no meta roles, allow access (public route)
  3. Get current user from request.user (authenticated via JwtStrategy or session)
  4. If no user, throw UnauthorizedException
  5. Check if user.role is in required roles array
  6. If not, throw ForbiddenException with message: `User role ${user.role} insufficient for this operation`
  7. If match, return true

**Auth/index.ts:**
- Export both Roles and RolesGuard for convenience

**RBAC-01 compliance:** @Roles() decorator complete  
**RBAC-02 compliance:** RolesGuard checks role before route execution

Tests should verify:
- Public routes (no @Roles) allow all authenticated users
- User with matching role gains access
- User without matching role gets ForbiddenException
- Unauthenticated requests get UnauthorizedException

**novels.controller.ts:**
Apply role guards to novel endpoints:
- `POST /novels` → `@UseGuards(RolesGuard) @Roles('ADMIN', 'AUTHOR')` (only admins/authors can create)
- `GET /novels` → Public or `@UseGuards(AuthGuard('jwt'))` (readable by authenticated users)
- `GET /novels/:id` → Public or authenticated
- `PATCH /novels/:id` → `@UseGuards(RolesGuard) @Roles('ADMIN', 'AUTHOR')` with owner check (can update own, or admin can update any)
- `DELETE /novels/:id` → `@UseGuards(RolesGuard) @Roles('ADMIN')` (admin only)

**users.controller.ts:**
Apply role guards to user endpoints:
- `PATCH /users/:id/role` → `@UseGuards(RolesGuard) @Roles('ADMIN')` (admin only can change roles)
- `GET /users` → `@UseGuards(RolesGuard) @Roles('ADMIN')` (admin only)
- `GET /users/:id` → Public or self-readable (user can see own, admin sees all)

**RBAC-03 compliance:** Novel CRUD operations protected  
**RBAC-04 compliance:** Role defaults to USER, admin-only endpoints require ADMIN

Code pattern:

```typescript
// In controller
import { Roles, RolesGuard } from '../auth';
import { UseGuards } from '@nestjs/common';

@Controller('novels')
export class NovelsController {

  @Roles('ADMIN', 'AUTHOR')

  @Delete(':id')
  @UseGuards(RolesGuard)
  @Roles('ADMIN')
  async remove(@Param('id', ParseIntPipe) id: number) { ... }
}
```
