# Current executeEtl Structure

Currently in etl-runner.ts:
1. Parallel load phase:
   ```typescript
   const [users, providers, wallets, vipRows, transactions, chapterRows] = await Promise.all([
     deps.loadUsers(),
     deps.loadProviders(),
     deps.loadWallets(),
     deps.loadVipRows(),
     deps.loadTransactions(),
     deps.loadPurchasedChapters(),
   ]);
   ```

2. Sequential migration phase:
   - migrateUsers (filters users, validates, upserts)
   - migrateVipSubscriptions (validates VIP data)
   - migrateTransactions (validates user references)
   - migratePurchasedChapters (upserts purchase history)

3. Return stats via mergeStats
