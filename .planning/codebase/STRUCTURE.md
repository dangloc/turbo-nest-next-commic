## Directory Purposes

**apps/api:**
- Purpose: Backend HTTP API service
- Contains: `src/` runtime code, `test/` e2e tests, Nest/TS/ESLint config
- Key files: `src/main.ts`, `src/app.module.ts`, `test/app.e2e-spec.ts`
- Subdirectories: `src/`, `test/`, generated `dist/`

**apps/web:**
- Purpose: Frontend Next.js application
- Contains: App Router pages, styles, static assets
- Key files: `app/layout.tsx`, `app/page.tsx`, `next.config.js`
- Subdirectories: `app/`, `public/`

**packages/ui:**
- Purpose: Shared UI components consumed by apps
- Contains: Component files (`button.tsx`, `card.tsx`, `code.tsx`)
- Key files: `src/button.tsx`, `package.json`
- Subdirectories: `src/`

**packages/eslint-config & packages/typescript-config:**
- Purpose: Shared baseline linting and type-checking config
- Contains: Flat ESLint config files and TS preset JSON files
- Key files: `packages/eslint-config/next.js`, `packages/typescript-config/base.json`

## Key File Locations

**Entry Points:**
- `apps/api/src/main.ts`: API server bootstrap and port binding
- `apps/web/app/page.tsx`: main web route component
- `apps/web/app/layout.tsx`: root app shell

**Configuration:**
- `package.json`: root scripts/workspaces/engine constraints
- `turbo.json`: task graph (`build`, `dev`, `lint`, `check-types`)
- `apps/api/eslint.config.mjs`, `apps/web/eslint.config.js`: app lint rules
- `apps/api/tsconfig.json`, `apps/web/tsconfig.json`: app type configs

**Core Logic:**
- `apps/api/src/app.controller.ts`: route handler
- `apps/api/src/app.service.ts`: service logic
- `packages/ui/src/*.tsx`: shared UI implementation

**Testing:**
- `apps/api/src/app.controller.spec.ts`: unit test
- `apps/api/test/app.e2e-spec.ts`: e2e test

**Documentation:**
- `README.md`: repository starter docs
- `.planning/codebase/*.md`: generated architecture reference docs

## Naming Conventions

**Files:**
- API files: lower-case dotted suffix style (`app.controller.ts`, `app.service.ts`)
- React component package files: lower-case (`button.tsx`, `card.tsx`, `code.tsx`)
- Test files: `*.spec.ts` and `*.e2e-spec.ts`

**Directories:**
- Workspace segment names are short lower-case (`apps`, `packages`, `web`, `api`)
- App Router route directories under `apps/web/app/`

**Special Patterns:**
- Shared package export mapping from `packages/ui/package.json` (`./*` -> `./src/*.tsx`)
- Flat ESLint configs in JS/MJS format

## Where to Add New Code

**New API feature:**
- Primary code: `apps/api/src/` (new module/controller/service)
- Tests: `apps/api/src/*.spec.ts` and/or `apps/api/test/*`
- Config changes: `apps/api/*.json` or root `turbo.json` as needed

**New frontend feature:**
- Pages/routes: `apps/web/app/<route>/page.tsx`
- Shared UI: `packages/ui/src/`
- Styles: `apps/web/app/*.css` or module CSS next to route

**New shared utility/config:**
- Linting: `packages/eslint-config/`
- TypeScript presets: `packages/typescript-config/`

## Special Directories

**apps/api/dist:**
- Purpose: Build output generated by Nest build
- Source: TypeScript compile output
- Committed: Currently present in workspace; should typically be ignored in monorepo source control

**apps/api/.git:**
- Purpose: Nested Git metadata indicates API may also be an independent repo
- Source: Existing local setup
- Committed: Git metadata itself is not committed in parent repo; this impacts commit workflows
