# Requirements: v1.8 Frontend Web Foundation

Milestone: v1.8
Defined: 2026-04-08
Completed: 2026-04-08
Core Value: Deliver the first complete web storefront experience on top of the shipped backend APIs.

## v1.8 Outcomes

### Frontend Init & Auth

- [x] WEB-01: Initialize and configure the web frontend foundation (routing/layout/feature folder conventions and environment-based API base URL wiring).
- [x] WEB-02: Implement shared API client utilities with typed request/response handling, auth-aware headers, and consistent error mapping.
- [x] AUTH-01: Integrate Google OAuth login flow in the web app against existing backend auth endpoints/session behavior.
- [x] AUTH-02: Provide authenticated user state bootstrapping (session load on app start, sign-in state persistence, sign-out UX path).

### Novel Discovery UI

- [x] DISC-01: Build homepage discovery UI that consumes discovery APIs and renders novel lists/cards with loading and empty states.
- [x] DISC-02: Build category/filter UI that supports pagination, sorting, and taxonomy filters mapped to discovery API query params.
- [x] DISC-03: Ensure discovery pages are shareable via URL query state (pagination/filter/sort reflected in the URL).

### Reader Experience UI

- [x] READ-01: Build chapter reading page that fetches and renders chapter content with novel/chapter navigation context.
- [x] READ-02: Trigger backend chapter view-count update flow when chapter view is opened.
- [x] READ-03: Persist and resume reading progress/history through existing reader personal APIs for authenticated users.

### Social UI

- [x] SOC-01: Render nested comment threads for chapter/novel scope using social comments APIs with deterministic parent-reply display.
- [x] SOC-02: Allow authenticated users to submit top-level comments and replies with inline validation/error states.
- [x] SOC-03: Support reaction toggles on comments with immediate UI update and eventual backend consistency.
