# Requirements Archive: v1.0 WordPress to NestJS Migration

**Archived:** 2026-04-07
**Core Value:** Preserve every financially or identity-sensitive record during the migration, with a repeatable ETL process and verifiable totals.

## v1 Requirements (Final Status)

### Validated

- [x] **DATA-01**: Prisma schema normalizes WordPress user, wallet, provider, VIP, transaction, and purchased chapter data into dedicated PostgreSQL tables. — Outcome: validated in Phase 1.
- [x] **DATA-02**: New user records retain the original WordPress password hash, user role, and stable legacy identifier mapping. — Outcome: validated in Phase 1.
- [x] **DATA-03**: Purchased chapter storage enforces uniqueness for the user/chapter pair and includes an index that supports large-scale lookup and deduplication. — Outcome: validated in Phase 1.
- [x] **MIG-01**: ETL script connects to the legacy MySQL database and the new PostgreSQL database using Node.js/TypeScript. — Outcome: validated in Phase 2.
- [x] **MIG-02**: ETL script migrates users, Google social login mappings, and wallet balances from the WordPress source tables. — Outcome: validated in Phase 2.
- [x] **MIG-03**: ETL script decodes serialized `vip_package` and `_purchased_chapters` values from `wp_usermeta` and maps them to normalized destination tables. — Outcome: validated in Phase 2.
- [x] **MIG-04**: ETL script writes purchased chapters in chunked batches with `createMany()` so migration remains safe for million-row volumes. — Outcome: validated in Phase 2.
- [x] **MIG-05**: ETL script migrates transaction history with legacy identifiers, amounts, timestamps, and content preserved where available. — Outcome: validated in Phase 2.
- [x] **MIG-06**: ETL script is idempotent or safely resumable so reruns do not create duplicate user, wallet, provider, VIP, or purchased chapter records. — Outcome: validated in Phase 2.
- [x] **VER-01**: Post-migration checks confirm total wallet balances match the legacy system. — Outcome: validated in Phase 3.
- [x] **VER-02**: Post-migration checks confirm purchased chapter counts match the successfully decoded WordPress source data. — Outcome: validated in Phase 3.
- [x] **VER-03**: Post-migration checks confirm social login mappings for Google users are preserved correctly. — Outcome: validated in Phase 3.
