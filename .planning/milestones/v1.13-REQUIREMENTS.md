## v1.13 Requirements (Completed)

### Legacy Financial Snapshot

- [x] **ETL-01:** Migration reads legacy user balance and VIP level from a single grouped `wp_usermeta` pivot per user.
  - **Acceptance Criteria:**
    - `wp_users.price` and `wp_users.vip_level_id` are ignored
    - SQL uses `MAX(CASE WHEN meta_key = '...' THEN meta_value END)` to extract `_user_balance`, `_user_vip_level_id`, and `_purchased_chapters`
  - **Completed:** 2026-04-12 (Phase 34)
  - **Phase Implementation:** src/etl/source-mysql-loaders.ts::loadUserFinancialSnapshots()
  - **Outcome:** ✅ Single grouped query reduces loading time from O(N) to O(1) per user

- [x] **ETL-02:** Migration transforms the serialized `_purchased_chapters` payload into Prisma-ready purchase rows.
  - **Acceptance Criteria:**
    - PHP-serialized values decode with `php-serialize`
    - Numeric-key objects are normalized
    - Mapped rows include actual target fields (`userId`, `chapterId`, `novelId`, `pricePaid`, `purchasedAt`)
  - **Completed:** 2026-04-12 (Phase 34)
  - **Phase Implementation:** src/etl/migrate-user-financials.ts
  - **Outcome:** ✅ Preserves legacy serialized data structure; handles edge cases (numeric keys, null values)

### Prisma Load Path

- [x] **ETL-03:** Migration updates target wallet/VIP state and bulk inserts purchase history with Prisma.
  - **Acceptance Criteria:**
    - Wallet/VIP updates and purchase history writes performed through Prisma repository methods
    - Chapter purchases use `createMany` with duplicate protection instead of per-row inserts
  - **Completed:** 2026-04-12 (Phase 34)
  - **Phase Implementation:** src/etl/prisma-repositories.ts (upsertFinancialSnapshot, createManyPurchasedChapters)
  - **Outcome:** ✅ Bulk inserts reduce database round-trips; `skipDuplicates` ensures safe reruns

## Key Technical Decisions

| Decision | Impact | Rationale |
|----------|--------|-----------|
| Single `wp_usermeta` pivot vs. per-user queries | Performance (O(1) vs O(N)) | Reduces query RTT and memory footprint |
| `php-serialize` library dependency | Reliability | Proven WordPress compatibility; already in package.json |
| Bulk `createMany` with `skipDuplicates` | Idempotency | Enables safe reruns without duplicate key conflicts |
| Decimal price precision for purchases | Data accuracy | Maintains exact legacy transaction amounts |
