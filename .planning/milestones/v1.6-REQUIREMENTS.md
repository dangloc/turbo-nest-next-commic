# Requirements Archive: v1.6 Core Reader API & Social Experience

**Milestone:** v1.6  
**Archived:** 2026-04-08  
**Status:** ✅ Complete

## v1.6 Requirements (All Completed)

### Reader Content APIs (READ)

- [x] **READ-01**: Public GET /novels endpoint supports pagination (`page`, `limit`) and returns total/count metadata.
  - **Outcome:** Implemented in Phase 16 with stable paginated response shape.
  - **Validated in:** Phase 16 discovery tests and full API regression.

- [x] **READ-02**: GET /novels supports sorting by `viewCount`, `updatedAt`, and `createdAt` with deterministic direction handling.
  - **Outcome:** Implemented with deterministic orderBy mapping.
  - **Validated in:** Phase 16 discovery tests.

- [x] **READ-03**: GET /novels supports filtering by taxonomy tags/category and publish status using query parameters.
  - **Outcome:** Implemented through taxonomy-based filtering.
  - **Adjusted:** Novel status is derived from taxonomy relations because the `Novel` model has no direct status column.

- [x] **READ-04**: GET /chapters/:id returns chapter payload and atomically increments both `chapter.viewCount` and parent `novel.viewCount`.
  - **Outcome:** Implemented with transaction-safe analytics updates.
  - **Validated in:** Phase 16 chapter tests.

- [x] **READ-05**: Authenticated bookmark APIs allow users to list/add/remove bookmarks while enforcing user ownership.
  - **Outcome:** Implemented under `/reader/me/bookmarks`.
  - **Validated in:** Phase 16 personal API tests.

- [x] **READ-06**: Authenticated reading history APIs allow upsert/get of user progress for resume behavior.
  - **Outcome:** Implemented under `/reader/me/reading-history`.
  - **Validated in:** Phase 16 personal API tests.

### Social Interaction APIs (SOC)

- [x] **SOC-01**: GET comments endpoint returns nested comment trees for novel/chapter scope including replies.
  - **Outcome:** Implemented with deterministic tree shaping.
  - **Validated in:** Phase 17 comments tests.

- [x] **SOC-02**: POST comment endpoint supports top-level comments and replies via optional `parentId`.
  - **Outcome:** Implemented with thread-aware comment creation.
  - **Validated in:** Phase 17 comments tests.

- [x] **SOC-03**: Comment creation validates scope (novel/chapter) and ownership fields from authenticated user.
  - **Outcome:** Implemented with scope normalization, parent inheritance, and authenticated ownership.
  - **Validated in:** Phase 17 comments tests.

- [x] **SOC-04**: Reaction toggle endpoint supports LIKE, HEART, HAHA, WOW, SAD, ANGRY enum values.
  - **Outcome:** Implemented under `/social/comments/:commentId/reactions`.
  - **Validated in:** Phase 17 reaction tests.

- [x] **SOC-05**: Reaction toggle respects uniqueness constraint (userId + commentId), preventing duplicate concurrent reactions.
  - **Outcome:** Implemented with unique-keyed toggle semantics and transactional safety.
  - **Validated in:** Phase 17 reaction tests.
