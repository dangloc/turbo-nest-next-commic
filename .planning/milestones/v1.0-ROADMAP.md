# Milestone v1.0: WordPress Migration Foundation

**Status:** SHIPPED 2026-04-07
**Phases:** 1-3
**Total Plans:** 7

## Overview

v1.0 delivered the migration foundation from legacy WordPress/MySQL to NestJS + PostgreSQL/Prisma with verification coverage for wallet balances, purchased chapters, and Google social mappings.

## Phases

### Phase 1: Schema Foundation

**Goal**: Design the PostgreSQL Prisma schema so legacy WordPress data lands in normalized, queryable tables without duplication.
**Depends on**: None
**Plans**: 1 plan

Plans:
- [x] 01-01: Prisma schema foundation and tooling

**Details:**
Defined normalized models (`User`, `UserProvider`, `Wallet`, `Transaction`, `VipSubscription`, `PurchasedChapter`) with legacy ID preservation and purchased chapter uniqueness/indexing.

### Phase 2: ETL Migration

**Goal**: Implement the migration job that extracts WordPress data, transforms serialized metadata, and loads normalized PostgreSQL records.
**Depends on**: Phase 1
**Plans**: 3 plans

Plans:
- [x] 02-01: ETL foundation contracts, parsers, and quarantine handling
- [x] 02-02: User/provider/wallet/VIP/transaction migration modules
- [x] 02-03: Purchased-chapter streaming and end-to-end runner wiring

**Details:**
Implemented one-time ETL with source/target connectivity, parse-safe handling of serialized metadata, resumable/idempotent writes, and chunked purchased chapter insertion.

### Phase 3: Verification

**Goal**: Prove the new database matches critical source totals and identity mappings after migration.
**Depends on**: Phase 2
**Plans**: 3 plans

Plans:
- [x] 03-01: Wallet total reconciliation command and discrepancy report artifact
- [x] 03-02: Purchased chapter reconciliation command with decode-aware mismatch reporting
- [x] 03-03: Google social mapping parity verification command and diff artifact

**Details:**
Verification commands and artifacts confirmed parity for wallet totals, purchased chapter counts (decoded source), and Google social mappings.

---

## Milestone Summary

**Key Decisions:**
- Preserve WordPress IDs and password hashes as canonical migration identity anchors.
- Keep ETL resumable/idempotent with upsert-based behavior and quarantine handling.
- Prioritize financial verification first in Phase 3 (wallet reconciliation before other checks).

**Issues Resolved:**
- Deterministic reconciliation reporting added for wallet, purchase, and social checks.
- Runtime artifact path normalization improved to stay stable across execution contexts.

**Issues Deferred:**
- Workspace scaffold/noise cleanup intentionally deferred per locked decision D-11.

**Technical Debt Incurred:**
- Missing per-plan summary docs for Phase 2 plans 01-03 at milestone closure time.
- No dedicated v1.0 milestone audit artifact was recorded before completion.
